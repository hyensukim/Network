* 네트워크 기초

네트워크란 컴퓨터와 같은 장치들이 두 대 이상 연결하고 서로 통신할 수 있는 것을 말한다.
또한, 연결할는 선이나 장치 같은 물리적인 측면부터 데이터를 주고받는 데에 필요한 메시지나 규약과 같은 물리적인 측면까지 모두 포함한, 통신의 전체 과정을 아우른다.

네트워크는 노드와 링크로 구성되어 있으며, 서로 연결되어 리소스를 공유한다.

노드는 서버, 라우터, 스위치 등의 네트워크 장치를 의미하며, 링크는 유선과 무선을 의미한다.

** 처리량과 지연시간

네트워크 구축 시에 고려해야 할 사항 중 처리량과 지연시간이 있다.

처리량은 링크 내에서 성공적으로 전달된 데이터의 양을 의미한다. 일반적으로 트래픽을 얼만큼 처리했는지로 나타낸다. 처리량이 높다는 말은 많은 트래픽을 처리함을 나타낸다.
단위는 bps(bit per second)를 사용한다. 초당 전송 비트 수를 의미한다.
처리량은 접속 트래픽, 네트워크 장치 간의 대역폭, 네트워크 에러, 장치 하드웨어 스펙에 의해서 달라진다.
- 대역폭 : 주어진 시간 동안 네트워크 연결을 통해 흐를 수 있는 최대 비트 수를 의미한다.

지연시간은 요청이 처리되는 시간을 의미한다. 즉, 한번의 요청이 서버에서 처리된 후 응답을 전달 받는데 까지 걸린 시간을 말한다.
지연시간은 매체타입, 패킷 크기, 라우터의 패킷 처리 시간에 영향을 받는다.
- 패킷 : 데이터 통신 시 특정 전달 형태 단위를 말한다. 한번에 많은 양을 한번에 보내는 것보다 패킷 단위로 전송하는 것이 지연시간을 줄이고 처리량을 높일 수 있다.

** 네트워크 토폴로지와 병목 현상

네트워크 토폴로지는 노드와 링크가 어떻게 배치되어 있는지에 대한 방식이자 연결 형태를 의미한다.
토폴로지가 중요한 이유는 병목현상을 찾을 때 중요한 기준이 되기 때문이다.

토폴로지의 종류로는 다음과 같다.
트리 토폴로지 : 계층형 토폴로지라고 하며 노드의 추가, 삭제가 쉽고, 특정 노드에 트래픽이 집중될 때 하위 노드에 영향을 끼칠 수 있다.

버스 토폴로지 : 중앙 통신 회선 하나에 여러 개의 노드가 연결되어 공유하는 네트워크 구성을 말하며 근거리 통신망(LAN)에서 사용한다. 설치 비용이 적고 신뢰성이 우수하지만, 스푸핑이 발생할 가능성이 있다.
- 스푸핑 : LAN 상에서 송신부의 패킷을 송신과 관련없는 호스트에 전달되도록 하는 것을 말한다. 이로 인해 패킷이 악의적인 노드로 전달될 수 있게될 위험이 있다.

스타 토폴로지 : 중앙에 있는 노드에 모두 연결된 네트워크 구성을 말한다. 노드 추가, 에러 탐지가 쉽고 패킷 간의 충돌 발생 가능성이 적다. 하지만 중앙 노드에 문제 발생 시에 전체 네트워크 사용이 불가하고, 설치 비용도 비싸다.

링형 토폴로지 : 각각의 노드가 양 옆의 두 노드와 연결하여 전체적으로 고리형태를 띈 망 구성 방식이다. 노드 수가 증가되어도 네트워크 상의 손실이 거의 없고, 충돌 발생 가능성이 적다. 고장부도 쉽게 찾을 수 있다는 장점이 있지만, 네트워크 구성 변경이 어렵고, 회선에 장애 발생 시 전체 네트워크에 영향을 미치게 된다.

메시 토폴로지 : 망형 토폴로지라고도 하며, 그물망 형태로 되어있다. 한 단말 장치에 장애가 발생하더라도 여러 경로가 존재하기 때문에 네트워크 사용이 가능하며, 트래픽 분산 처리도 가능하지만, 노드 추가가 어렵고 구축 비용과 운용 비용이 고가이다.

병목현상 : 전체 시스템의 성능이나 용량이 하나의 구성 요소로 인해 제한을 받는 현상을 의미한다.

** 네트워크 분류
규모에 따른 분류
LAN : Local Area Network, 근거리 통신망을 의미하며, 좁은 공간에서 운영된다. 전송 속도가 빠르고 혼잡하지 않다.
MAN : Metropolitan Area Network, 대도시 지역 네트워크망을 의미하며, 넓은 지역에서 운영된다. 전송 속도는 평균이며, LAN보다 더 많이 혼잡하다.
WAN : Wide Area Network, 국가 또는 대륙 단위의 더 넓은 지역에서 운영되며, 전송 속도는 낮으며, MAN보다 더 혼잡하다.
속도 : LAN > MAN > WAN
규모 : WAN > MAN > LAN
복잡도 : WAN > MAN > LAN

** 네트워크 성능 분석 명령어
네트워크 성능 문제로 인해서 장애가 발생하는 경우도 있으며, 주요 원인은 다음과 같다.
- 네트워크 대역폭
- 네트워크 토폴로지
- 서버 CPU, 메모리 사용량
- 비효율적인 네트워크 구성

이처럼 장애가 발생했을 때, 네트워크 성능을 분석하기 위한 명령어들이 존재한다.

ping(Packet INternet Groper)
네트워크 상태를 확인하려는 대상 노드를 향해 일정 크기의 패킷을 전송하는 명령어이다. 해당 노드의 패킷 수신 상태와 도달하기 까지의 시간을 알 수 있다. ping은 TCP/IP 프로토콜 중에 ICMP 프로토콜을 통해서 동작한다. 실행은 `ping [IP 주소 또는 도메인 주소]` 로 실행한다. 

netstat
접속되어 있는 서비스들의 네트워크 상태를 표시하는 데 사용되며 네트워크 접속, 라우팅 테이블, 네트워크 프로토콜 등 리스트를 보여준다. 주로 해당 서비스의 포트가 열려있는지 확인 시 사용된다.

nslookup
DNS에 관련된 내용을 확인하기 위해 사용되는 명령어이다. 특정 도메인에 매핑된 IP를 확인하기 위해서 사용한다.

tracert
목적지 노드까지 네트워크 경로 확인 시 사용하는 명령어이다. 어느 구간에서 응답 시간이 느려지는지 등을 확인할 수 있다.

** 네트워크 프로토콜 표준화
네트워크 프로토콜이란 다른 장치들끼리 데이터를 주고받기 위해 설정된 공통된 인터페이스를 의미한다. 프로토콜은 개인이 정하는 것이 아니라 IEEE 또는 IETF 라는 표준화 단체가 정한다. 예를 들어, 웹을 접속하는 경우에 HTTP 라는 프로토콜을 기반으로 웹 서비스 내 데이터를 주고 받을 수 있다.
